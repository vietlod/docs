# Data Administration and Pipelines

This page is for administrators or Ecodata operators. Regular users do not need these steps to download indicators.

## Main Admin Modules

| Module | Role |
| --- | --- |
| Admin Dashboard | Monitor system, data, permissions, and pipelines. |
| GSO Admin | Run or import GSO data, manage aliases, and check extraction. |
| Customs Admin | Manage Customs report batches and import status. |
| Stock Admin | Manage stock data, financial statements, and event calendar. |
| Translation Jobs | Translate metadata, variable labels, and indicator descriptions when needed. |
| Chat Providers | Configure AI providers for Chat. |
| Export Requests | Monitor full export requests and processing status. |

## Operating Principles

1. Do not bypass service layers when adding a new pipeline.
2. Do not edit production data directly without backup.
3. On a shared VPS, check ports, Nginx, and Docker before deploy.
4. Do not use compose files or scripts marked legacy/dead in the internal KB.
5. Every data change needs UI and API smoke tests.

## Checks After Ingestion

- Indicator and data point counts change as expected.
- Search and filter return correct results.
- Preview displays correct metadata.
- Export creates files within the selected scope.
- Dashboard, GSO, Customs, Survey, and Stocks pages have no console or API errors.
