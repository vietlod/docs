# Indicator Groups

In Ecodata, an indicator is more than a variable name. Each indicator should be understood together with its source, coverage, frequency, unit, metadata, methodology, and export permissions.

## Indicator Metadata

| Field | Meaning |
| --- | --- |
| Indicator name | Display name shown to users. |
| Code or short code | Identifier used for search, copy, export, or AI Chat discussion. |
| Source | Original data source or ingestion pipeline. |
| Category | Topic group such as macro, trade, labor, governance, survey, or stock. |
| Coverage | Countries, provinces, firms, commodities, and available year range. |
| Frequency | Annual, quarterly, monthly, or report-period frequency. |
| Unit | Measurement unit, price basis, percent, index, VND, or USD. |
| Completeness | Data availability within the selected scope. |
| Methodology | Definition and calculation method from the source. |
| Citation | How to cite the data in research outputs. |
| Quality notes | Notes on missing values, methodology changes, or alternate sources. |

## Indicator Groups by Module

| Module | How to select indicators |
| --- | --- |
| Dashboard | Filter by keyword, category, source, frequency, year, and completeness; open details before export. |
| GSO Explorer | Choose a dataset, browse the catalog, select indicators, and view province-year pivots. |
| Customs Explorer | Select report type, reporting period, commodity, country, province, or transport mode. |
| Survey Explorer | Filter survey, province, year, indicator group, and indicator details. |
| VHLSS Variable Hub | Select waves, datasets, variables, preview records, then choose merge, panel, or pooled output. |
| Stock Hub | Select a symbol and inspect price, financial statements, notes, or events. |

## Rules for Combining Indicators

1. Do not mix units of analysis unless aggregation or merging is explicit.
2. Do not mix monthly, quarterly, and annual data without frequency conversion.
3. Do not mix currencies, current prices, and constant prices without normalization.
4. Use metadata to compare indicators with similar names across sources.
5. Use Preview to detect missing values before modeling.

## AI Chat Workflow for Indicator Selection

1. Describe the research question, unit of analysis, period, and expected model.
2. Ask AI Chat to suggest dependent variables, explanatory variables, controls, and sources.
3. Search those suggestions in Dashboard or the relevant module.
4. Copy selected indicator short codes back to AI Chat to validate the model logic.
5. If coverage is incomplete, ask for proxy indicators or alternate sources.
