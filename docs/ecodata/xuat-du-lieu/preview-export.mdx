# Preview và Export dữ liệu

Preview và Export là bước cuối trong workflow khai thác dữ liệu. Người dùng nên xem đây là bước kiểm định dữ liệu, không chỉ là thao tác tải file.

## Preview kiểm tra gì

| Hạng mục | Câu hỏi cần trả lời |
| --- | --- |
| Scope | Dữ liệu có đúng indicators, quốc gia/tỉnh/symbol/hàng hóa không? |
| Time | Khoảng năm/kỳ báo cáo có đủ không? |
| Unit | Đơn vị đo có thống nhất không? |
| Missing | Có thiếu dữ liệu theo năm hoặc entity quan trọng không? |
| Metadata | Có source, definition, methodology, citation, notes không? |
| Structure | Bảng đang ở dạng long, wide, pivot, panel hay record-level? |

## Export trong Dashboard

1. Chọn indicators.
2. Chọn countries, provinces hoặc nhóm quốc gia nếu có.
3. Chọn year range.
4. Chọn cấu trúc bảng.
5. Bật include metadata nếu cần tái lập nghiên cứu.
6. Preview.
7. Download hoặc theo dõi request export đầy đủ nếu tài khoản có quyền.

## Export trong các module chuyên biệt

| Module | Kiểu export |
| --- | --- |
| GSO Explorer | CSV hoặc Excel, thường có sheet Data và Desc/metadata. |
| Customs Explorer | CSV/Excel theo report và dimension đã lọc. |
| Survey Explorer | CSV cho indicators cấp tỉnh. |
| VHLSS Variable Hub | Dataset theo mode single-wave, panel hoặc pooled; full export phụ thuộc quyền. |
| Stock Hub | Bảng giá, BCTC, notes hoặc events theo symbol/kỳ. |

## Quyền tài khoản

Ứng dụng phân biệt preview và full export theo entitlement. Tài khoản miễn phí thường có quyền preview để kiểm tra dữ liệu mẫu; tài khoản có gói phù hợp có thể tạo export đầy đủ.

## Checklist file sau khi tải

- File dữ liệu có đủ số dòng/số cột kỳ vọng.
- Metadata hoặc sheet mô tả đi kèm được lưu cùng dữ liệu.
- Tên indicators và mã short code được giữ lại.
- Đơn vị đo và nguồn không bị mất khi chuyển sang phần mềm phân tích.
- Nếu dùng nhiều nguồn, có file ghi lại cách merge và transform.
